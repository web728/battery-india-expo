# Developer Handoff — India Battery International Show 2026 Website

This is a complete Next.js 16 website. It **runs immediately with zero configuration** —
forms work, all pages render — you're only configuring things to move from "demo mode" to
"live with a real database and email."

Full details are in [README.md](README.md). This file is just the ordered checklist to get
from zip file to deployed site.

## 1. Install & run locally

```bash
npm install
npm run dev
```

Open the printed localhost URL. Click around — forms submit successfully and log to your
terminal (no database needed yet).

> **Node.js 20+** required (project uses Next.js 16 / Tailwind v4).

## 2. Set environment variables (all optional, see table in README)

```bash
cp .env.example .env.local
```

Fill in what the client has provided (Supabase project keys, Resend API key, analytics IDs).
None of these are required for the site to run — only to persist form submissions to a real
database and send real emails instead of console-logging them.

## 3. Connect Supabase (recommended before going live)

1. Create a project at [supabase.com](https://supabase.com).
2. Run `supabase/schema.sql` in the Supabase SQL editor — creates every table, RLS policy, and enum.
3. Copy the Project URL / anon key / service role key into `.env.local` (see step 2).
4. Create an admin account (sign up via `/login`'s password-reset flow, or directly in the
   Supabase dashboard), then set that user's `profiles.role` to `super_admin` to unlock `/admin`.

## 4. Connect email (Resend)

1. Create an account at [resend.com](https://resend.com), verify a sending domain.
2. Add `RESEND_API_KEY` and `EMAIL_FROM_ADDRESS` to `.env.local` / your hosting provider's env vars.

## 5. Deploy

**Vercel (recommended, zero config):**
1. Push this repo to GitHub/GitLab.
2. Import into Vercel.
3. Add the env vars from `.env.example` under Project Settings → Environment Variables.
4. Deploy.

**Any other Node host:**
```bash
npm run build
npm run start
```

## 6. Before announcing the site is live

- [ ] Replace placeholder statistics in `src/lib/content/stats.ts` once the client confirms numbers.
- [ ] Add real exhibitors to `exhibitors` table (or `src/lib/content/exhibitors.ts`) only once
      bookings are confirmed — do not add placeholder companies.
- [ ] Have the client's legal counsel review the six draft legal pages (Privacy Policy, Terms,
      Registration Terms, Cancellation Policy, Cookies, Accessibility) — currently marked
      "Draft for Legal Review" and set to `noindex`. Once approved, remove the `robots: { index: false }`
      line from each page's `metadata` export.
- [ ] Set `NEXT_PUBLIC_SITE_URL` to the production domain if different from the default in
      `src/lib/site-config.ts`.
- [ ] Add real favicon/app icons (currently the default Next.js `favicon.ico`).
- [ ] Set analytics IDs (`NEXT_PUBLIC_GA_MEASUREMENT_ID`, GTM, Meta Pixel, LinkedIn) if required at launch.

## What's NOT included in this package

- `node_modules/` — regenerated by `npm install`.
- `.next/` — regenerated by `npm run build`.
- The raw, full-resolution source photos (16–22MB each) — only the optimized, web-ready
  versions in `public/images/` are included, since those are what the site actually serves.
- Any `.env.local` — you'll create this from `.env.example` with real credentials.

## Questions about the content/scope

This site's content is intentionally scoped to only what's in the official event brochure —
no invented conference agenda, speakers, or B2B networking platform, no fabricated stand
pricing. See the "Content policy" section of [README.md](README.md) before adding new pages
or "coming soon" sections.
